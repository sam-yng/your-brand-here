# Sunday Service and Common Thread

Status: queued after Northline architecture proof.

## Objective

Complete the range proof with two visibly distinct fictional systems while preserving
shared accessibility, content honesty, progressive enhancement, and bundle limits.

## Dependencies

- Northline exit confirms study scoping pattern.
- Study font subset of EXT-09 available, or temporary fallbacks explicitly tracked.

## Scope and order

1. Build Sunday Service namespace, hero, truthful fictional-release language, shift
   content, feature cards, build note, and CTA. Preserve Anton/Karla type, platform
   chips, coming-2027 language, key-art placeholder, rotating eight-ray sun SVG,
   Breakfast/Lunch/Dinner themes, anti-dark-pattern note, and owned palette.
2. Implement shift controls with all content server-rendered and programmatic selected
   state. Ship the wishlist control as an explicitly labelled fictional demonstrator:
   activation may explain the intended interaction, but submits nothing, never claims
   “On your wishlist,” and never uses confirmed-success green. A real successful state
   requires a separately approved provider/acceptance boundary and tests; never
   reproduce prototype-only optimism.
3. Build Common Thread namespace, hero, six-product lookbook, URL-backed category
   filters, sizing/care content, build note, and CTA. Preserve Anybody/Public Sans
   type, wide/condensed word forms, three collection bars, every name/price/fabric/
   category/XS–4XL chip, and owned palette.
4. Implement filters as ordinary links first, then enhance without losing URL state.
5. Implement sizing/care trigger, region/drawer semantics, predictable focus, close
   behavior, repairs, measurements, mill, care, returns content, and complete print
   CSS. Add Escape only if final implementation behaves as a modal overlay.
6. Add route-owned CreativeWork metadata inputs and focused tests without sharing
   visual layout components across studies.
